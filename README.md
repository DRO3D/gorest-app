# GoREST Client

Тестовое задание: React + TypeScript + Consta UI клиент для https://gorest.co.in/.

## Стек

- **React 18** + **TypeScript**
- **Consta UI Kit** (@consta/uikit, @consta/icons)
- **react-router-dom** v6 для навигации
- **Vite** как сборщик
- Управление состоянием — нативный React Context + хуки (внешняя стейт-библиотека для такого объёма не нужна)

## Запуск

```bash
npm install
npm run dev
```

Открыть http://localhost:5173

## Получение токена

Зарегистрироваться и получить access token:
https://gorest.co.in/my-account/access-tokens

## Функциональность

### Главная (`/`)
- Поле для access token, валидация на непустоту
- После сохранения токена — переключатель режимов (Пользователи / Посты) и кнопка "Открыть"
- Токен сохраняется в `localStorage`, сбрасывается кнопкой "Выйти"

### Список пользователей (`/users`)
- Таблица: Имя, Фамилия, Email
  - API отдаёт `name` целиком, поэтому имя/фамилия разделяются по первому пробелу
- Пагинация:
  - Номера страниц (с эллипсисом для больших списков)
  - Кнопки "Предыдущая" / "Следующая"
  - Селект количества на странице (10/25/50)
- Клик по строке → карточка пользователя

### Список постов (`/posts`)
- Таблица: ID, Заголовок
- Аналогичная пагинация
- Клик по строке → карточка поста

### Карточка пользователя (`/users/:id`)
- Имя, статус, пол, ID, email
- Список всех постов пользователя (клик ведёт на карточку поста)
- Кнопка возврата к списку

### Карточка поста (`/posts/:id`)
- Заголовок, тело поста
- Кнопка "Открыть автора"
- Все комментарии к посту
- Кнопка возврата к списку

## Совместимость

Проверено на современных Chromium-браузерах (Яндекс.Браузер, Edge). Layout рассчитан на разрешения 1024–1920 (max-width контента 1600px, гибкая раскладка).

## Структура

```
src/
├── api/client.ts          — обёртка над fetch + GoREST endpoints, чтение x-pagination-* заголовков
├── components/
│   └── Pagination.tsx      — переиспользуемый компонент пагинации
├── hooks/
│   ├── useAuth.tsx         — контекст токена
│   └── usePaginatedList.ts — хук для загрузки списков с пагинацией
├── pages/
│   ├── HomePage.tsx
│   ├── UsersPage.tsx
│   ├── PostsPage.tsx
│   ├── UserCardPage.tsx
│   └── PostCardPage.tsx
├── types/api.ts            — типы User/Post/Comment
├── App.tsx                 — роутинг
├── main.tsx                — точка входа, Theme provider
└── styles.css
```

## Замечания по API

GoREST v2 отдаёт мета-информацию пагинации в HTTP-заголовках (`x-pagination-total`, `x-pagination-pages`, `x-pagination-limit`, `x-pagination-page`), поэтому она читается из `response.headers`, а не из тела.
